TauViz-CT

TauViz-CT is a lightweight visualization demo for 3D porous microstructures using speed-colored streamlines derived from v = −∇C (a pseudo-flow field computed from a 3D concentration map). The rendering overlays a semi-transparent solid skeleton and optional “comet-tail” glow points, producing an intuitive “flowing inside pores” effect for presentations and method demonstrations.

⬇ Download streamlines_tool.py

The script reads a 3D binary volume (0 = pore, 1 = solid) and a 3D concentration map (Cmap; TIFF or MAT), builds a pseudo-vector field from v = −∇C, generates streamlines inside pores, and exports an MP4 video.

How to run locally

You can run TauViz-CT on your own 3D CT data as long as you provide: (i) a 3D binary volume (0 = pore, 1 = solid) and (ii) a 3D concentration map (Cmap). This is intended for qualitative visualization (not a CFD solver), and the visual pattern depends on how your Cmap is defined.

1) Install dependencies (recommended: in a clean Python environment): 

python -m pip install numpy matplotlib tifffile pillow imageio imageio-ffmpeg scikit-image scipy

2) Put inputs next to the script (or edit file paths in the CONFIG block inside the script), then run: 

python streamlines_tool.py

3) Tune parameters in the CONFIG block (near the top of the script). Common settings:

  • BINARY_PATH, CMAP_PATH, OUTPUT_MP4: input/output files.
  • VOXEL_UM: voxel size (sets aspect ratio; wrong value can make the shape look “boxy”).
  • N_LINES, STEP, N_STEPS, MIN_POINTS: streamline density/length (more lines = richer but slower).
  • ENABLE_GLOW, GLOW_WINDOW, GLOW_SPEED, GLOW_SIZE: the blue “comet-tail” dots (turn off if you want lines only).
  • DS_MESH, DS_FIELD, SMOOTH_SIGMA: speed/quality trade-offs (higher downsampling = faster but coarser).

Tip: if you get very few streamlines, try increasing N_LINES, decreasing MIN_POINTS, and/or increasing SMOOTH_SIGMA.

Notes

  • This is a visualization demo intended for qualitative illustration (not a CFD solver).
  • The pseudo-flow field is derived from −∇C; interpretation depends on your physical meaning of the Cmap boundary conditions.
  • If you use this demo in slides/papers, please cite your CT/transport workflow (e.g., TauFactor/Avizo/your pipeline) accordingly.
